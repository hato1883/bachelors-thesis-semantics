\chapter{Background}
\label{cha:background}
\noindent
This chapter establishes the theoretical foundation necessary for the thesis.
We begin by specifying the \While language, which serves as the primary object
of study throughout this work. This provides the concrete syntax required to
explore the formal semantics of programming languages—the mathematical bridge
between a program's notation and its behavior. We then delve into Operational
Semantics, detailing both the Natural (big-step) and Structural (small-step)
approaches, followed by the primary proof techniques used in their analysis.
Finally, we introduce the Lean 4 proof assistant, detailing the mechanisms used
to translate these textbook definitions into machine-checked proofs.

\section{Formal Semantics of Programming Languages}
\label{background:sec:formal_semantics_pl}
\todo{What?}
The formal semantics of a programming language is a rigorous mathematical
framework used to describe the behavior of programs \missingcitation. While a language's syntax
defines the rules for constructing valid programs (the "grammar"), its semantics
provides the "meaning"—specifying exactly how a computer should execute those
instructions \missingcitation.

\todo{Why?}
In many industrial settings, the semantics of a language are described using
natural language manuals \missingcitation. However, natural language is often prone to ambiguity,
leading to inconsistent implementations across different compilers or interpreters \missingcitation.
Formal semantics eliminates this ambiguity by using mathematical structures—such
as sets, functions, and logic to define program execution \missingcitation.

By separating how an instruction is written (the syntax) such as
\begin{While_Code}
    if (x > 0) then {x = x} else {x = -x}
\end{While_Code} % \noindent
from the meaning (the semantics) such as "if x is less than 0 then negate the value, otherwise do nothing" it is possible to
create a deeper and more detailed understanding (\hampus{this statment has no basis}) \missingcitation of the programming language
and therby the programs written.


\todo{How?}
Formal semantics are typically expressed through three main approaches:
Denotational (modeling programs as mathematical functions),
Axiomatic (using logic to define preconditions and postconditions),
and Operational \missingcitation. This thesis focuses on the Operational approach, which describes
how a program transition from one state to another during execution.

Operational semantics describes the meaning of a program by specifying how it
executes on an abstract machine. Unlike other semantic approaches, it is
"execution-oriented," focusing on the transitions between program states.
A state transition is typically represented as a relation between a
configuration—consisting of a statement and the current state—and a resulting
state or a subsequent configuration.
