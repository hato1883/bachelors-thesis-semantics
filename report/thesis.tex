To be able to verify a specify program it can be necessary to verify the programming language it is written in as well, or at least partial aspects of it.

\todo{Then be even more specific using Programming langauge semantics} \noindent
This is where Programming Language Semantics is used,

\todo{Finaly introduce the two methods studied, written and mechanized proofs} \noindent
but modern proofs are constructed in 2 different ways, one method formalizes the proof in a natural language such as english using established terms.
Whilst some proofs lean on the mathematical proofing methods to ensure a less ambiguous proof to allow the proof to be verified by automatic tooling such as proof Assistants.
This thesis aims to evaluate the differences between these two methods, and try to uncover any ambiguity created in the proof techniques used, such as implicit assumptions.

\section{Research Question and Goals}
\todo{What is this thesis about? (Mechanizing semantic proofs)} \noindent
In this thesis we will explore what, if any implicit assumptions must be made explicit when mechanizing a written programming language semantics formal proof of into Lean 4.
We will also evaluate if there exists any subtleties in the formal proofs that is being left to the reader to prove or understand for the proof to be valid.  
%\lipsum[1]

\todo{Why does it matter? (Informal proofs hide complexity)} \noindent
This can be helpful to highlight aspects of proofs that can often be left to the readers to understand or assume, whilst in a mechanized proof is stated explicitly.
Therein give writers a pointer into what they should highlight when constructing proofs to decrease any possible ambiguity introduced by using a written form of the proof.

\section{Programming Language Semantics: Four Approaches}
\label{background:sec:pls}
\todo{Introduce Programming Language Semantics and the 2 (4 if we got time) diffrent approaches}
There are four major approaches to construct semantics for a programming language, of which we will examine proofs of two of them.
In this section we will do a quick overview of thier diffrences.
First approach is called "Natural Semantics" often refered to as "Big-Step Semnatics" and often shortend to "ns" which we will explore in \autoref{background:subsec:ns}.
Secound approach is called "Structural Operational Semantics" which is refered to as "Small-Step Semantics" and shortend to "sos" explored in \autoref{background:subsec:sos}.

The third approach which will not be exaxmined in this thesis is "Axiomatic Semantics".

Lastly we have "Denotational Semantics" which also wont be studied in this thesis.

We will also do a quick overview of the toy language "While" introduced by the litterature "Semantics with Applications An Appetizer" in \autoref{background:subsec:while_lang}.
